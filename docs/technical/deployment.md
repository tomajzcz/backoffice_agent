## 2. Vercel Deployment

### Setup

1. Connect the GitHub repository to Vercel.
2. Set all required environment variables in the Vercel dashboard (Settings > Environment Variables).
3. Deploy. Vercel auto-deploys from the `main` branch on every push.

### Build Configuration

| Setting | Value | Notes |
|---------|-------|-------|
| Framework | Next.js | Auto-detected by Vercel |
| Build command | `next build` | Default |
| Output directory | `.next/` | Default |
| Node version | 20.x | LTS |
| Install command | `npm install` | Triggers `postinstall` hook, which runs `prisma generate` |

The `postinstall` script in `package.json` runs `prisma generate` automatically, so the Prisma client is always up to date after install.

### Runtime Configuration

| Setting | Value | File |
|---------|-------|------|
| `serverExternalPackages` | `["pdfkit"]` | `next.config.ts` |
| `maxDuration` (chat) | 120s | `app/api/chat/route.ts` |
| `maxDuration` (weekly-report) | 120s | `app/api/cron/weekly-report/route.ts` |
| `maxDuration` (monitoring) | 60s | `app/api/cron/monitoring/route.ts` |
| `maxDuration` (reminder-calls) | 60s | `app/api/cron/daily-reminder-calls/route.ts` |

The 120-second timeout on the chat route requires the **Vercel Pro** plan (Hobby plan limit is 60 seconds).

### Cron Jobs (vercel.json)

Defined in `vercel.json`:

| Job | Schedule | Endpoint | Description |
|-----|----------|----------|-------------|
| Market Monitoring | `0 5 * * 1-5` | `/api/cron/monitoring` | Scrape Sreality + Bezrealitky (weekdays at 5:00 UTC) |
| Reminder Calls | `0 5 * * *` | `/api/cron/daily-reminder-calls` | ElevenLabs voice calls for today's showings (daily at 5:00 UTC) |
| Weekly Report | `0 7 * * 1` | `/api/cron/weekly-report` | Executive PPTX report + email (Monday 7:00 UTC / 9:00 CEST) |

All cron endpoints validate the `Authorization: Bearer ${CRON_SECRET}` header. Vercel injects the `CRON_SECRET` value automatically when invoking cron functions.

```json
{
  "crons": [
    { "path": "/api/cron/monitoring", "schedule": "0 5 * * 1-5" },
    { "path": "/api/cron/daily-reminder-calls", "schedule": "0 5 * * *" },
    { "path": "/api/cron/weekly-report", "schedule": "0 7 * * 1" }
  ]
}
```

## 3. Database (Neon)

### Setup

1. Create a Neon project at [neon.tech](https://neon.tech).
2. Create a database within the project.
3. Copy the **pooled** connection string and set it as `DATABASE_URL`.
4. Copy the **direct** connection string and set it as `DIRECT_URL`.

The pooled connection uses Neon's connection pooler (PgBouncer) for runtime queries. The direct connection is required for Prisma migrations, which need a persistent connection.

### Initial Setup

```bash
npm run db:generate    # Generate Prisma client
npm run db:migrate     # Apply all migrations
npm run db:seed        # Populate with seed data (45 clients, 150 leads, 55 properties, etc.)
```

### Migrations

After changing `prisma/schema.prisma`:

```bash
npm run db:migrate     # Creates a new migration file + applies it
npm run db:generate    # Regenerates Prisma client types
```

To start from scratch (destructive):

```bash
npm run db:reset       # Drops all tables, re-migrates, re-seeds
```

## 4. Local Development

### Prerequisites

- Node.js 20+
- npm 9+
- PostgreSQL (local instance or Neon remote database)

### Quick Start

```bash
git clone <repo-url>
cd backoffice_agent
npm install                    # Installs deps + generates Prisma client via postinstall
cp .env.example .env.local     # Fill in required values (DATABASE_URL, DIRECT_URL, ANTHROPIC_API_KEY)
npm run db:migrate             # Apply migrations
npm run db:seed                # Populate test data
npm run dev                    # Start dev server at http://localhost:3000
```

### Available Scripts

| Script | Description |
|--------|-------------|
| `npm run dev` | Start Next.js development server |
| `npm run build` | Production build |
| `npm run lint` | Run ESLint |
| `npm run db:generate` | Generate Prisma client from schema |
| `npm run db:migrate` | Create + apply pending migrations |
| `npm run db:seed` | Seed database with test data |
| `npm run db:reset` | Reset DB + re-seed (destructive) |
| `npm run db:studio` | Open Prisma Studio GUI |

### Prisma Studio

```bash
npm run db:studio    # Opens visual DB browser at http://localhost:5555
```

Useful for inspecting and editing data during development without writing queries.

## 5. Graceful Degradation

When optional integrations are not configured, the application degrades gracefully rather than failing:

| Missing Config | Behavior |
|---------------|----------|
| Google credentials | Calendar and Gmail tools return error messages to the agent |
| Twilio credentials | SMS sending is skipped; failure captured as `smsError` in tool results |
| ElevenLabs credentials | Voice calls are skipped; status logged as `FAILED` or `SKIPPED` |
| n8n credentials | Webhook integration is disabled |
| `CRON_SECRET` | Cron endpoints reject all requests with 401 Unauthorized |

Core functionality (chat, analytics, CRUD operations, chart rendering) works with only the three required environment variables.

## 7. Known Deployment Considerations

| Consideration | Detail |
|--------------|--------|
| PDFKit bundling | Must be listed in `serverExternalPackages` in `next.config.ts`; it cannot be bundled by the Next.js compiler |
| Prisma client generation | Handled automatically at `postinstall` (`npm install` triggers `prisma generate`) |
| Google OAuth refresh token | Requires manual setup outside the application; there is no built-in OAuth consent flow |
| Neon cold starts | First request after idle period may experience additional latency due to Neon compute spin-up |
| Vercel Pro plan | Required for 120-second function timeout on the chat route (Hobby plan caps at 60 seconds) |
| Cron timezone | Vercel cron schedules run in UTC; configured times account for the CET/CEST offset |
| In-memory file storage | PPTX and PDF download tokens are stored in memory and lost on serverless cold starts; files should be downloaded promptly after generation |
| Single Google account | Calendar and Gmail are bound to one Google account via refresh token; multi-tenant OAuth is not implemented |
| Scraper fragility | HTML structure changes on Sreality or Bezrealitky will break parsers; monitor for parse errors and update selectors as needed |
